# INA228 是啥，拿它干嘛

## 一句话

**它是块专门"数电"的芯片** —— 同时测电压、电流、功率、累积能耗 + 温度，通过 I²C 告诉主机豆豆现在用了多少电、每分每秒消耗多少焦耳。英文叫 **power monitor / current sense amplifier**。

## 物理上测什么

INA228 装在**高压端**（电池和负载之间），同时测两个量：

### 1. VBUS — 母线电压
直接测 IN+（或 IN-）对 GND 的电位，精度 195.3125 µV/LSB，量程 0–85V。
**用途**：实时监测电池电压，掉到阈值以下触发自动降级 / 回充。

### 2. VSHUNT — 分流电阻两端差压
IN+ 和 IN- 之间串着一个已知阻值的**分流电阻 (shunt)**，所有负载电流都流过它。通过测 shunt 两端微小压降（几十 mV），按欧姆定律 `I = V / R_shunt` 反推电流。20-bit ADC，LSB 可以到 nV 级。
**用途**：电流监测。

### 直接算出来的复合量
芯片内部硬件自带乘法器和积分器，把上面两个原始量加工成：

| 寄存器 | 内容 | 公式 | 用途 |
|-------|------|------|------|
| CURRENT | 瞬时电流 | VSHUNT × 标度 | 实时功率、异常检测 |
| POWER | 瞬时功率 | VBUS × CURRENT × 标度 | 即时功耗 |
| ENERGY | 累积能量 | ∫ POWER dt | 续航估算、任务能耗结算 |
| CHARGE | 累积电量 | ∫ CURRENT dt | 电量百分比、SoC |
| DIETEMP | 芯片温度 | 内部温度传感器 | 过热保护 |

**能耗积分全部在芯片内部做**，主机周期性读一次就行，不用自己在 CPU 上积分，没漏采样 / 没浮点漂移。

## 在豆豆里的典型角色

INA228 是豆豆的"电表 + 电流表 + 燃油表"，装在 **6S 电池到主电源总线** 之间：

```
电池 (6S LiPo)
    │
    ● IN+    ────────────
    │                    ↓
   shunt 2mΩ          电流穿过
    │                    ↑
    ● IN-    ────────────
    │
    ├──→ 电机驱动 (12 个 damiao，最耗电大头)
    ├──→ Jetson Orin Nano (计算平台)
    ├──→ RealSense / ReSpeaker / 舵机 / LED
    └──→ 其他负载
```

主机 (Jetson) 通过 I²C 读 INA228，持续拿到：

- **电池电压** → 低电告警、回充决策
- **瞬时电流 / 功率** → 观察"哪个动作最耗电"、堵转检测
- **累积能量** → 这次任务耗了多少 mWh、还能跑多久
- **温度** → 芯片自身过热保护（电池温度要另外传感器）

## 具体能做的几件事

### 实时遥测
```
[电压] 24.37V    (6S, 健康)
[电流]  8.2 A    (行走中)
[功率] 199.8 W   (对照 motor 扭矩 + 头部推理负载)
[温度]  46.3 °C  (INA228 die)
```

把这些打到 Grafana / MQTT / ROS topic，豆豆每秒的能耗状态一目了然。

### 电池电量估计（SoC）
- 6S LiPo 满电 25.2V、断电保护 19.8V
- 读 VBUS + 查电压-SoC 曲线 → 粗估剩余百分比
- 用 CHARGE 累积电量 + 初始电量 → 精确 SoC（需要周期性用 VBUS 校正）

### 故障检测
- 电流突增（>X 秒持续 >Y A）→ **堵转 / 撞墙 / 短路**
- 电压骤降（>Z V 瞬时）→ **电池 sag 异常，可能要限流**
- 功率远超预期（同一动作耗电变多）→ **机械卡阻 / 轴承磨损**

### 续航规划
- 记录每种动作的平均功率（站立 / 行走 / 跑步 / 跳跃）
- 结合当前剩余能量 → 算"还能跑多少分钟 / 多少米"

## 能做 / 不能做

### 能做
- 高精度电流测量（20-bit ADC，2 mΩ shunt 下分辨率 ~200 µA）
- 高精度电压测量（195 µV/LSB）
- 硬件积分累积能量，无漏采
- 多种 ADC 模式（连续 / 触发 / 平均）
- 可编程阈值告警（ALERT 引脚触发中断）

### 不能做
- **电池内阻 / 温度 / 单电芯均衡** —— 这是 BMS 的事，INA228 只测总线 I/V
- **电机相电流** —— 只测总母线电流，想知道"哪个电机此刻吃了多少电"需要每个电机自带电流环（DM4310-P 的 MIT 帧里自带 τ）
- **负端电流** —— INA228 测单向 high-side (IN+ 到 IN-)，反向充电场景需要特殊配置

## 为啥选 INA228（对比 INA219 / INA226）

| 特性 | INA219 | INA226 | **INA228** |
|------|--------|--------|-----------|
| VBUS 量程 | 26V | 36V | **85V** |
| ADC 分辨率 | 12-bit | 16-bit | **20-bit** |
| VSHUNT LSB | 10 µV | 2.5 µV | **312 nV / 78 nV** |
| 硬件能量积分 | ✗ | ✗ | **✓** |
| 价格 | $1.5 | $3 | **~$5** |

豆豆 6S LiPo 峰值 25.2V，INA219 勉强够；INA226 留余量；**INA228 是"过度充裕" 但是换来了 20-bit 分辨率和硬件能耗积分**，后者做续航估计和功耗分析省心很多。

TI 把 INA228 定位在"电动自行车 / 工业设备 / 服务器电源"，用在四足机器人上完全匹配。

## 和其它传感器的配合

- **damiao 电机自带扭矩 τ 反馈** → 每电机出力
- **INA228 总电流 I** → 整机能耗
- 两者交叉验证：`Σ(τ × ω) / η_motor ≈ P_mechanical`，和 `P_bus = V × I` 对比可以估出电气效率 η
- **电池电压 VBUS** + **电流 I** → LiPo 内阻估计（闲置 vs 负载电压差）
- 长期记录 → "同一动作在不同电量下的执行时间"，辅助任务调度
