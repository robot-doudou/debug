# BMI088 是啥，拿它干嘛

## 一句话

**它是块测"自己在空间里怎么动"的芯片** —— 直接测加速度和角速度，让机器人"知道"自己的身体在做什么动作、朝什么方向。英文叫 **IMU (Inertial Measurement Unit)，惯性测量单元**。

## 物理上测什么

两颗芯片封装在一起，各司其职：

### 加速度计 (ACC) —— 测"力"

- 三轴加速度，单位 m/s²
- 包含两类信号叠加：
  1. **重力** —— 静止时地球总在你身上施加 ~9.8 m/s² 的向下加速度
  2. **运动加速度** —— 设备被推、被撞、起步、刹车时产生的

### 陀螺仪 (GYR) —— 测"转"

- 三轴角速度，单位 dps（度/秒）或 rad/s
- 每个轴独立报告"绕自己转得多快"
- 静止时应该 ≈ 0

## 直接能算出来的东西

从这两样原始数据，派生出机器人里最关键的几个状态：

| 派生量 | 怎么来 | 用途 |
|-------|-------|------|
| **姿态 (roll / pitch)** | ACC 静止时重力方向 → 倾角 | 机器人身体是不是平的、侧倾多少 |
| **yaw (航向)** | GYR 积分 | 机器人转了多少度 |
| **角速度** | GYR 直接读 | 机器人旋转快慢，PID 闭环必备 |
| **线加速度** | ACC 减掉重力分量 | 运动冲击、碰撞检测 |
| **步态相位** | ACC 的振动特征 | 四足落脚/抬腿节拍 |
| **跌倒检测** | 姿态角突然 > 阈值 | 保护性关闭 |

## 在豆豆里的典型角色

四足机器人的 IMU 是**平衡控制闭环的眼睛**：

```
   IMU (姿态 + 角速度)
        ↓ 读数 1000 Hz
   平衡控制器
        ↓ 下发扭矩
   12 个关节电机 (你的 damiao)
        ↓ 腿动作
   地面反作用力
        ↺ 反馈回 IMU
```

没有 IMU，机器人就是个"不知道自己是正是歪"的瞎子 —— 摔倒也察觉不到，更别提站稳和走。**IMU 决定了机器人的"平衡反射"**。

## 能做 / 不能做

### 能做（短时间尺度，秒级）

- 实时姿态、身体角速度（高带宽、低延迟）
- 动态冲击检测、跌倒保护
- 辅助预测下一步状态

### 不能做（长时间尺度，分钟级）

- **位置估计** —— ACC 积分两次算位置，误差指数级爆炸，几秒就漂出几米
- **绝对航向 (yaw)** —— 没磁力计，GYR 积分会慢慢跑偏，几分钟可能偏几度到几十度
- **在剧烈运动中准确估 roll/pitch** —— 运动加速度会"污染"重力分量

这些长时间漂移问题必须靠**传感器融合**来修正：

- ACC 修 GYR 的 roll/pitch 漂移（互补/卡尔曼/Madgwick 滤波）
- 视觉里程计（豆豆有的 **D435i**）修 yaw 和位置
- 磁力计修 yaw（豆豆目前没配）

## 为啥选 BMI088（相比其它 IMU）

BMI088 在竞品里（MPU6050、ICM42688 等）的特色：

- **双 die 设计**：ACC 和 GYR 是物理上分开的两颗芯片封在一起 → 机械振动耦合低，尤其抗电机振动
- **专为机器人/无人机设计**：Bosch 官方定位就是 "robotics and drones"
- **温度稳定性好**：零偏漂移 < 2 mg/K
- **便宜** (~$7)

DJI 的大疆无人机、不少 ROS 项目、Unitree 的四足都在用 BMI088 或它的表亲。

## 和 D435i 内置 IMU 的分工

D435i 里那颗 BMI055（实际上是 BMI088 的前代）IMU 只跟相机同轴，装在相机"头部"。如果豆豆的相机是在头部舵机上活动的，**它的 IMU 跟着头部晃**，不能代表身体。

所以分工：

- **BMI088（本项目调的这颗）** → 装在躯干/底盘，测**身体**状态
- **D435i 内置 IMU** → 跟相机一起动，做**视觉 + 惯性里程计 (VIO)** 时用

两颗都接进融合算法（ROS 里通常是 `robot_localization` 的 EKF），得到鲁棒的姿态和位姿估计。
